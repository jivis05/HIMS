# WeaveHealth HIMS - Frontend Architecture & Structure

This document outlines the architecture, directory structure, and technology stack of the Frontend portion of the WeaveHealth Hospital Information Management System (HIMS).

## 1. Technology Stack

*   **Core Library:** React 19 (via Vite)
*   **Routing:** React Router DOM
*   **Styling:** Tailwind CSS (utility-first CSS) + Framer Motion (animations) + Lucide React (icons)
*   **HTTP Client:** Axios
*   **State Management:** React Context API (custom contexts)
*   **Internationalization:** i18next & react-i18next
*   **Build Tool:** Vite

## 2. Top-Level Directory Structure

The frontend application is contained within the `frontend/` directory.

```text
frontend/
├── dist/                  # Production build output (generated by Vite)
├── node_modules/          # Project dependencies
├── public/                # Static assets served directly (e.g., favicon, icons)
├── src/                   # Main application source code
│   ├── assets/            # Local static files like images and fonts
│   ├── components/        # Reusable UI and functional components
│   ├── context/           # React Context for global state management
│   ├── data/              # Static mock data or configuration files
│   ├── hooks/             # Custom React hooks
│   ├── pages/             # Route-level components representing full views
│   ├── services/          # API integration and external service logic
│   ├── App.jsx            # Main application component and route definitions
│   ├── App.css            # Global CSS for the App component
│   ├── index.css          # Main Tailwind imports and global styles
│   └── main.jsx           # Application entry point (ReactDOM rendering)
├── .env                   # Environment variables for local development
├── eslint.config.js       # ESLint configuration for code linting
├── index.html             # The main HTML template for the SPA
├── package.json           # Dependencies and NPM scripts
├── tailwind.config.js     # Tailwind CSS configuration and theme settings
└── vite.config.js         # Vite bundler configuration
```

## 3. Deep Dive into `src/` Architecture

### 3.1. Pages (`src/pages/`)
The `pages` directory contains top-level components that correspond to specific routes in the application. The system is heavily role-based, reflecting different dashboards:

*   **Dashboards:**
    *   `AdminDashboard.jsx`: Facility management.
    *   `SuperAdminDashboard.jsx`: Global system oversight.
    *   `DoctorDashboard.jsx`: Clinical workflows and patient consultations.
    *   `PatientDashboard.jsx` & `PatientEMR.jsx`: Patient portal and Electronic Medical Records access.
    *   `ReceptionistDashboard.jsx`: Appointments and front-desk management.
    *   `LabTechnicianDashboard.jsx`: Laboratory tests and report management.
    *   `PharmacyDashboard.jsx`: Pharmacy inventory and prescriptions.
    *   `BloodBankDashboard.jsx`: Blood stock and donor management.
    *   `NurseDashboard.jsx`: Inpatient and ward management.
*   **Authentication & Profiles:**
    *   `Login.jsx` & `Register.jsx`: User authentication.
    *   `Profile.jsx`: User profile settings.
    *   `OrgRegister.jsx` & `OrgDashboard.jsx`: Organization/Hospital onboarding.
*   **Management Views:**
    *   `StaffManagement.jsx`: Handling hospital personnel.

### 3.2. Components (`src/components/`)
Reusable UI pieces broken down into specific domains:
*   `common/`: Generic UI components (buttons, modals, tables, inputs).
*   `layout/`: Structural components (Sidebar, Navbar, Footer, PageWrappers).
*   `auth/`: Components specifically related to authentication flows (e.g., 2FA setup modals).

### 3.3. Services (`src/services/`)
This layer handles all communication with the backend API.
*   Uses `axios` instances configured with base URLs and interceptors.
*   Interceptors automatically inject JWT authentication tokens into request headers and handle global API errors (like redirecting on a 401 Unauthorized response).

### 3.4. Context (`src/context/`)
Global state is managed via React Context. Typical contexts might include:
*   **AuthContext:** Manages the current user's login state, JWT token, and role information.
*   **ThemeContext:** Manages UI state like light/dark mode.

## 4. Application Flow & Routing
1.  **Entry:** `main.jsx` mounts the app to the DOM and wraps it in necessary providers (Contexts, Router, i18n).
2.  **Routing:** `App.jsx` defines the routing logic. It uses React Router to map URLs to components in the `pages/` directory.
3.  **Guards:** Routes are protected based on authentication state and user roles (e.g., a patient cannot access the `AdminDashboard`).

## 5. Available Scripts
Defined in `package.json`:
*   `npm run dev`: Starts the Vite local development server.
*   `npm run build`: Bundles the app for production into the `dist/` folder.
*   `npm run preview`: Locally previews the production build.
*   `npm run lint`: Runs ESLint to check for code quality issues.
